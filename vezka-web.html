VEZKA Web
Platform

Designing a high-conversion, responsive product website to showcase the mobile app’s ecosystem, optimize the digital onboarding funnel, and drive user downloads.

View Figma Prototype

The Context

As an extension of the mobile banking application, the Vezka Web Platform was designed as a comprehensive corporate and marketing website. Rather than an operational transaction hub, this multi-page platform serves as the product’s digital flagship. It features tailored landing pages for both Personal and Business accounts—directly targeting freelancers and independent workers—alongside a dedicated Help Center for FAQs to reduce user friction. As the sole designer, I focused on scaling the app's visual identity into a responsive web layout, structuring cohesive navigation and grid systems across multiple pages to optimize user acquisition and engagement.

Information Architecture & Sitemap

[ Organizing the System ]

To translate Vezka’s mobile ecosystem into a comprehensive website, I structured a clean, multi-page information architecture. The main goal was to ensure that personal solutions, business tools, and the dedicated Help Center were effortlessly accessible, providing a frictionless navigation system that segments user needs while strategically driving mobile app downloads.

[ Key Nav Nodes ]
1. Home & Product Landing: Overview of the mobile app ecosystem, key digital banking features, and app download links.
2. Personal Banking: Highlights of individual account benefits, card management options, and day-to-day financial features.
3. Business & Freelance: Dedicated segment featuring tailored solutions, account perks, and transaction overviews for independent workers.
4. Help & Support Center: Directory of interactive FAQs, search tools, and a direct portal to connect with human agents.

Wireframing: Paper to Digital

[ 1. Paper Wireframes ]

Initial Ideation

Using pen and paper, I sketched several layout variations for the web dashboard. The main focus was analyzing how the sidebar navigation expands/collapses and designing flexible tables to house large financial data strings without compromising readability.

Sketches were shared with peers to validate standard desktop layout patterns and check side-by-side card proportions.

[ 2. Digital Wireframes ]

Refining Content Hierarchy & Responsiveness

Moving to Figma, I designed digital wireframes focused on simplifying navigation and highlighting the most-used features first. The primary objective was to ensure maximum layout clarity and accessibility for different user types across all ages.

Based on feedback, I adjusted the visual hierarchy, refined spacing margins, and optimized button contrast to improve overall scanability and readability on smaller mobile screen variations.

Direct Call-to-Action (CTA): A prominent homepage button invites users to open a free account directly, prompting rapid action without intermediate subpages.

FAQ Carousel: A sleek horizontal carousel organizes frequently asked questions on the landing page, allowing users to find immediate answers to common concerns.

Interactive Prototype

[ Testing Flow Logic ]

Before jumping into high-fidelity visual styling, I mapped and connected the wireframe screens in Figma to build an interactive, clickable prototype. This flow allows us to test the logic of onboarding procedures, client management, and search navigations, creating a solid blueprint to evaluate user paths before development.

Final Visuals

The final interface translates VEZKA's identity into a responsive web layout designed for e-commerce utility. By balancing a high-contrast color scheme, structured typography, and clear alignment grids, the layout supports user conversion goals while maintaining a clean visual hierarchy across all devices.
